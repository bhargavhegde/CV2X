◆ CMS_WSMP_SUBSCRIBE_WILDCARD

#define CMS_WSMP_SUBSCRIBE_WILDCARD   0UL

Wildcard value for the WSMP subscribe function's psid parameter.

When this value is used as a key for the WSMP subscribe function, all received WSMP packets will trigger a notification.

Subscribe for received WSMP messages.

Upon successful subscription, when a WSMP message is received with a matching PSID, the ITS software stack sends a notification to the subscribed client with some metadata and the raw message.

Parameters
sessionClient session
psidPSID to subscribe for
callback_fNotification callback function
ctxUser context
[out]subs_idSubscription ID output

true in case of error

bool cms_wsmp_rx_unsubscribe ( const cms_session_t *  session,
cms_subs_id_t  subs_id 
)

Unsubscribe from received WSMP messages.

Upon successful unsubscription, when a WSMP message is received with a matching PSID, the ITS software stack does not send a notification to the subscribed client any more. Messages with different subscribed PSID will still be sent as a notification.

Parameters
sessionClient session
subs_idSubscription ID, retrieved by cms_wsmp_rx_subscribe
Returns
true in case of error
