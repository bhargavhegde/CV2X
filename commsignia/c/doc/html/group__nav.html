Enumerator

Nav fix is initialized according to configuration at startup.

Nav fix can be set via cms_nav_set_manual. Timestamp is updated to system time if not explicitly set.

◆ cms_nav_set_manual()

bool cms_nav_set_manual ( const cms_session_t *  session,
const cms_nav_set_manual_t *  nav_set,
void *  unused_out 
)

Set the navigation fix for manual positioning mode.

Note
auto-update functionality is not yet supported.

About is_valid field: nav_set.nav_fix.is_valid field is automatically calculated by the stack, in case of latitude or longitude have NA value: is_valid will be overwritten to false, any other case: is_valid from the nav_set struct will be used

In case of invalid or zero timestamp: nav_set.nav_fix.timestamp is invalid timestamp in case of 0 or CMS_UTC_TIMESTAMP_NA. In case of invalid timestamp the timestamp will be overwritten by current system time when the fix is used.

Expiration of the navigational fix: After cms_nav_set_manual function with valid timestamp usage the stack is expecting continuous refresh of navigation data. Make sure you are calling cms_nav_set_manual with at least 1 HZ (recommended: 10HZ) frequency if you want to use this function with valid timestamp.

Setting system time: In case of valid timestamp and adjustSystemTime (config parameter) is enabled: system time and leapseconds value is set to the values from nav_set If you want to set only the system time please use cms_nav_set_time.

sessionClient session
[in]nav_setNavigation fix input
[out]unused_outUnused parameter
Returns
true in case of error
