const cms_wsa_reg_client_srv_param_t *  params,
cms_wsa_reg_client_srv_out_t *  out 
)

Function to register a client's need and priority for a WSA service.

Upon successful registration, a client ID is returned, which can be used for deregistration. Successful registration can result in a channel change of the V2X hardware, and making already registered services available/unavailable, depending on the registered services' priority and the currently active (received) WSAs. In case of reprioritization of a WSA service, cms_wsa_deregister shall be called first, then cms_wsa_register again.

Parameters
sessionClient session
paramsRegister client service parameters
[out]outRegister client service output parameters
Returns
true in case of error

cms_wsa_service_status_notify_f  wsa_service_status_notify,
void *  ctx,
cms_subs_id_t *  subs_id

WSA service status notification subscribe.

After successful subscription, when there is a change in the availability of a service identified by its PSID, i.e. if there is a frequency change, a notification is sent for each service that became available or unavailable.

This notification is sent for every PSID for which the status has changed regardless of current service clients, i.e. even if there are no registered service clients a received WSA will trigger this notification for every contained PSID.

Parameters
sessionClient session
psidNon p-encoded PSID identifying the service
wsa_service_status_notifyWSA service status notification callback
ctxUser context
[out]subs_idSubscription ID output
Returns
true in case of error
